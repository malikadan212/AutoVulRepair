           About MemFix Memory Deallocation Repair
        

        

          MemFix uses a sophisticated 3-phase pipeline based on static analysis and SAT solving 
          (Lee, Hong & Oh - ESEC/FSE 2018) to automatically repair memory deallocation errors:
        

        
    
          
  • Memory Leak (CWE-401): Inserts free() calls at optimal program points
    • 
          
  • Double-Free (CWE-415): Removes duplicate free() statements and ensures single deallocation
    • 
          
  • Use-After-Free (CWE-416): Moves free() calls to safe locations after last use
    • 
        

        

          How MemFix Works
          

            
Phase 1: CFG Construction & Pre-Analysis

            
Builds Control Flow Graph and runs points-to/alias analyses

            
            
Phase 2: Typestate Analysis

            
Tracks object states through program execution using transfer functions τ (patch update) and φ (points-to update)

            
            
Phase 3: SAT-Based Exact Cover

            
Formulates the repair as a Boolean SAT problem with constraints:

            
    
              
  • φ1: No memory leaks (every object covered)
    • 
              
  • φ2: No double-frees (each object covered at most once)
    • 
              
  • φ3: No use-after-free from patch ordering
    • 
            

            

              Key Feature: MemFix removes ALL existing free() statements 
              and reinserts them at optimal locations, ensuring correctness.
